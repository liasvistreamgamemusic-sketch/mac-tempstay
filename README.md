# Perch

画面の端に出てくる「一時的なシェルフ」で、ドラッグ中のファイルを置いておける macOS メニューバーアプリ。Yoink 系のワークフローを Apple Silicon の Mac 向けに再現したオープンソース実装です。

ファイルを別のウィンドウやアプリへ移すとき、一度 Perch のシェルフに「ちょい置き」してから運べるので、ウィンドウの切り替えやドラッグの持ち替えが要らなくなります。

## 特長

- **端へドラッグで自動表示** — ファイルを画面端へドラッグするとシェルフがスッと現れます。
- **何でも置ける** — ファイル / フォルダ / 画像 / テキスト / URL を一時保管。
- **ドラッグして取り出す** — シェルフのアイテムを目的のアプリ・フォルダへドラッグするだけ。
- **コピーして保持** — 置いたファイルはコピーされるので、元ファイルを移動・削除しても大丈夫。
- **クリップボードを取り込む** — ショートカット一発で今のクリップボードをシェルフへ。
- **グローバルショートカット** — 表示/非表示・クリップボード取り込み・全消去を自由に割り当て。
- **左右どちらの端にも配置可能**、自動で隠す/隠さないも設定可能。
- **メニューバー常駐**（Dock には出ません）。アイコンはシェルフの中身に応じて変化。
- **自動アップデート** — 新しいバージョンが出ると通知し、その場でダウンロード＆置き換え。

## 動作環境

- macOS 14 (Sonoma) 以降
- Apple Silicon / Intel どちらも（ユニバーサルではなくビルド環境依存。CI は arm64 でビルド）

## インストール

1. [Releases](https://github.com/liasvistreamgamemusic-sketch/mac-tempstay/releases) から最新の `Perch-x.y.z.dmg` をダウンロード。
2. DMG を開き、`Perch.app` を `Applications` へドラッグ。
3. 初回起動時、署名は ad-hoc のため Gatekeeper の警告が出ることがあります。`システム設定 > プライバシーとセキュリティ` から「このまま開く」を選んでください。

> 注: メニューバーにのみ表示されます。Dock アイコンはありません。

## 使い方

| 操作 | 動作 |
| --- | --- |
| ファイルを画面端へドラッグ | シェルフが自動で出現（設定で ON/OFF） |
| シェルフへドロップ | アイテムとして一時保管 |
| アイテムをドラッグして外へ | 目的の場所へコピー／移動 |
| アイテムをダブルクリック | ファイル/URL を開く（テキストはクリップボードへコピー） |
| アイテムにホバー → ✕ | 個別に削除 |
| ヘッダーのゴミ箱 | 全消去 |

### 既定のショートカット

| アクション | ショートカット |
| --- | --- |
| シェルフの表示/非表示 | ⌃⌥V |
| クリップボードをシェルフへ | ⌃⌥C |
| シェルフを空にする | ⌃⌥X |

ショートカットは設定の「ショートカット」タブから変更できます（⌘ または ⌃ を含む組み合わせのみ）。

## 開発

Swift Package Manager ベース。UI は AppKit + SwiftUI のハイブリッド。

```bash
# ビルド & 起動（開発用、未署名・バンドル外）
swift run

# テスト
swift test

# 配布用 .app を生成（アイコン生成 + ad-hoc 署名込み）
./scripts/build_app.sh 0.1.0

# DMG を生成
./scripts/make_dmg.sh 0.1.0
```

### アーキテクチャ

```
Sources/Perch/
  App/          アプリのライフサイクルと配線 (AppDelegate, ShelfCoordinator)
  Models/       値型のドメインモデル (ShelfItem, ShelfEdge, HotkeyAction)
  Store/        状態の単一ソース (ShelfStore, SettingsStore, AppSettings)
  Persistence/  ディスク上のアイテム管理 (ItemStorage)
  Shelf/        シェルフのウィンドウと SwiftUI ビュー
  DragIn/       ドロップ取り込みと端トリガー検知
  DragOut/      シェルフからのドラッグ提供 (NSItemProvider)
  StatusBar/    メニューバー項目とアイコン
  Hotkeys/      グローバルショートカット (Carbon RegisterEventHotKey)
  Settings/     設定 UI とショートカット記録
  Update/       GitHub Releases ベースの自動アップデート
  Support/      共通定数・ログ・ログイン項目
```

各レイヤーは一方向に依存し（UI → Coordinator → Store → Persistence）、状態は必ず Store を経由します。機能追加は基本的に「モデルに 1 ケース足す」「Store にメソッドを足す」だけで UI と配線が追従するように設計しています。

## リリースの仕組み

- `main` への push で CI（ビルド + テスト）が走ります。
- `vX.Y.Z` タグを push すると、Release ワークフローが DMG をビルドして GitHub Release に添付します。
- アプリは起動時に最新リリースを確認し、新しければ通知・その場でアップデートできます。

```bash
# リリース手順
git tag v0.1.0
git push origin v0.1.0
```
